## Naming
- Test files: [name].test.ts or [name].spec.ts
- Test suites: describe('ComponentName', ...)
- Tests: it('should do something when condition', ...)
- Setup: beforeEach, beforeAll (not before)

## Structure
- Colocate: component.tsx + component.test.tsx
- E2E tests: tests/e2e/[feature]/
- Fixtures: tests/fixtures/
- Test utilities: tests/utils/

## Patterns
- Arrange/Act/Assert pattern
- Test behavior, not implementation
- One assertion per test (or cohesive assertions)
- Use factories for test data
- Mock external dependencies (APIs, databases)
- Reset state between tests

## Quality Standards
- >70% code coverage for critical code
- All tests pass before merging
- No flaky tests (fix or delete)
- Clear test names (no "it works" or "test 1")
- Tests run in <10 seconds (unit), <1min (integration)
