## Naming
- Components: PascalCase (e.g., UserProfile, LoginButton)
- Hooks: camelCase with 'use' prefix (e.g., useAuth, useLocalStorage)
- Files: kebab-case matching component name (user-profile.tsx)
- CSS classes: Use Tailwind utilities, avoid custom CSS unless necessary

## Structure
- Colocate tests with components (component.tsx, component.test.tsx)
- UI components in src/components/ui/ (shadcn/ui)
- Feature components in src/components/[feature]/
- Shared hooks in src/hooks/
- Types in same file or adjacent .types.ts

## Patterns
- ALWAYS use components from src/components/ui/ instead of custom divs
- Never use React.memo, useMemo, useCallback (React Compiler handles it)
- Form validation: Zod schemas + react-hook-form
- Data fetching: Use custom hooks (useInvoke, useInvokeMutation)
- Error boundaries around major features
- Loading states for async operations

## Quality Standards
- Desktop-density sizing: h-8 default (32px), h-6 small (24px)
- Never use h-9, h-10, or size="lg" for standard controls
- 8px spacing grid (p-2=8px, p-4=16px, p-6=24px)
- Use CSS variables from globals.css, never hardcoded colors
- Transitions: 100-200ms, use transition-colors or transition-all
- Shadow: shadow-sm max for elevation
- Icons: Lucide only, 16px default
