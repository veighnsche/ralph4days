## Naming
- Containers: kebab-case (app-frontend, app-backend)
- CI jobs: kebab-case, verb-first (build-frontend, test-backend)
- Scripts: kebab-case (deploy-staging.sh, backup-db.sh)
- Env vars: SCREAMING_SNAKE_CASE (DATABASE_URL, API_KEY)

## Structure
- Dockerfiles: Dockerfile (or Dockerfile.[variant])
- CI configs: .github/workflows/, .gitlab-ci.yml
- IaC: terraform/, cloudformation/
- Scripts: scripts/ or bin/
- Configs: config/ or .config/

## Patterns
- Multi-stage Docker builds (builder + runtime)
- .dockerignore to exclude unnecessary files
- Health check endpoints (/health, /ready)
- Graceful shutdown (SIGTERM handling)
- Secrets via environment variables, never hardcoded
- Use official base images, pin versions

## Quality Standards
- CI pipelines fail fast (linting before tests before build)
- Caching for faster builds (Docker layers, npm cache)
- Parallel jobs where possible
- Clear job names and outputs
- Deployment gates (require approval for production)
- Automated rollback on health check failure
