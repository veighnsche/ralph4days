## Naming
- Services: PascalCase suffix (UserService, AuthService)
- Routes: kebab-case (e.g., /api/users, /api/auth/login)
- Functions: camelCase, verb-first (getUserById, validateEmail)
- Constants: SCREAMING_SNAKE_CASE (MAX_RETRIES, DB_TIMEOUT)

## Structure
- Routes: src/routes/[resource].ts
- Controllers: src/controllers/[resource]-controller.ts
- Services: src/services/[resource]-service.ts
- Models: src/models/[resource].ts
- Middleware: src/middleware/[name].ts
- Utilities: src/utils/[category].ts

## Patterns
- Validate at API boundaries (never trust input)
- Use dependency injection for testability
- Separate business logic from HTTP concerns
- Use Result types for error handling (avoid throwing)
- Structured logging with context (user ID, request ID, etc.)
- Idempotent operations where possible

## Quality Standards
- HTTP status codes: 2xx success, 4xx client error, 5xx server error
- Error responses include: code, message, details (never leak internals)
- API versioning via URL (/api/v1/) or header
- Rate limiting for public endpoints
- Request/response validation with schemas
- Comprehensive integration tests for critical paths
