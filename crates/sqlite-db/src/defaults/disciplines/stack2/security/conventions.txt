## Naming
- Security functions: verb-first (validateInput, sanitizeHtml, hashPassword)
- Secret env vars: PREFIX_SECRET (e.g., DB_PASSWORD, API_SECRET_KEY)

## Patterns
- Validate inputs at API boundaries (schemas, whitelists)
- Sanitize outputs before rendering (prevent XSS)
- Use parameterized queries (never string concatenation)
- Hash passwords with salt (bcrypt, argon2)
- Store tokens securely (httpOnly cookies for web)
- Implement rate limiting on auth endpoints
- Use CSRF tokens for state-changing operations

## Quality Standards
- All passwords hashed, never stored plaintext
- Secrets in environment variables, never in code
- Security headers configured (CSP, HSTS, etc.)
- Dependencies scanned for vulnerabilities
- Authentication rate limited (prevent brute force)
- Authorization checked on every protected resource
